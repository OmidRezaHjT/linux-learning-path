# Processes

## Boot Process
English: **1)Motherboard runs POST (Power-On Self-Test)/2)Motherboard loads the bootloader/3)Bootloader loads the Linux kernel./4)Kernel initializes the system and root filesystem./5)Init program starts services (web, GUI, networking, etc.)**
**فارسی: **۱) مادربورد تست اولیه (POST) را اجرا می‌کند./۲)مادربورد بوت‌لودر را بارگذاری می‌کند./۳) بوت‌لودر کرنل لینوکس را بارگذاری می‌کند./۴) کرنل سیستم و فایل‌سیستم ریشه را مقداردهی می‌کند./۵) برنامه init سرویس‌ها را اجرا می‌کند (وب، رابط گرافیکی، شبکه و …).

## Bootloader
English: **Bootloader sets up minimal hardware and runs the OS; on GNU/Linux, GRUB is used, which can run any program but usually starts the OS**
**فارسی: **فارسی: بوت‌لودر حداقل سخت‌افزار لازم را آماده کرده و سیستم‌عامل را اجرا می‌کند؛ در GNU/Linux معمولاً از GRUB استفاده می‌شود که می‌تواند هر برنامه‌ای را اجرا کند اما معمولاً سیستم‌عامل را راه‌اندازی می‌کند.

## Kernel
English: **The kernel, the core of the OS, is loaded by the bootloader into memory and can receive parameters from GRUB for modes or settings**
**فارسی: **کرنل، هسته سیستم‌عامل، توسط بوت‌لودر در حافظه بارگذاری می‌شود و می‌تواند پارامترهایی از GRUB برای حالت‌ها یا تنظیمات دریافت کند

## init
English: **After kernel initialization, the init system starts all necessary services, daemons, and programs in order, managing dependencies. Admins can control services later. Common init systems are SysVinit (older Unix style), Upstart (event-based, mostly replaced), and Systemd (modern, parallel service startup). Init always has PID 1**
**فارسی: **بعد از مقداردهی کرنل، سیستم init سرویس‌ها، دیمون‌ها و برنامه‌های لازم را به ترتیب و با مدیریت وابستگی اجرا می‌کند. مدیر سیستم می‌تواند سرویس‌ها را بعداً کنترل کند. انواع رایج init شامل SysVinit (قدیمی)، Upstart (مبتنی بر رویداد، بیشتر جایگزین شده) و Systemd (مدرن، اجرای موازی سرویس‌ها) هستند. فرآیند init همیشه PID ۱ دارد

## systemd
English: **systemd is the modern init system used in most Linux distros; it manages services, dependencies, and can start processes in parallel**
**فارسی: **systemd سیستم init مدرن است که در بیشتر توزیع‌های لینوکس استفاده می‌شود؛ سرویس‌ها و وابستگی‌ها را مدیریت کرده و می‌تواند پردازش‌ها را به صورت موازی اجرا کند

## systemctl
English: **systemctl is the main command to manage services in systemd (start, stop, enable, disable, check status)**
**فارسی: **دستور systemctl فرمان اصلی برای مدیریت سرویس‌ها در systemd است (اجرا، توقف، فعال‌سازی، غیرفعال‌سازی، بررسی وضعیت)

## SysV
English: **SysVinit is the older init system. Its control scripts are located in /etc/init.d/ and are similar to general bash scripts**
**فارسی: **SysVinit سیستم init قدیمی‌تر است. فایل‌های کنترلی آن در /etc/init.d/ قرار دارند و شبیه اسکریپت‌های عادی bash هستند

## runlevels
English: **Runlevels define which tasks or services are active in a Linux system at a given state, like different stages of system operation**
**فارسی: **Runlevelها مشخص می‌کنند در هر وضعیت یک سیستم لینوکس چه کارها یا سرویس‌هایی فعال هستند، مثل مراحل مختلف عملکرد سیستم

You can switch to special targets in systemd:
rescue: mounts local filesystems, no networking, root only (maintenance).
emergency: only root filesystem in read-only, no networking, root only.
reboot: restarts the system.
halt: stops all processes and CPU.
poweroff: like halt but also sends ACPI shutdown signal.


## SysV runlevels
English: **On SysVinit, different stages of system operation could be defined using runlevels**
**فارسی: **در SysVinit می‌شد مراحل مختلف عملکرد سیستم را با استفاده از runlevelها تعریف کرد.
REDHAT:
0- Shutdown
1- Single-user mode (recovery); Also called S or s
2- Multi-user without networking
3- Multi-user with networking
4- to be customized by the admin
5- Multi-user with networking and graphics
6- Reboot
DEBIAN:
0- Shutdown
1- Single-user mode
2- Multi-user mode with graphics
6- Reboot


## Grub
English: **GRUB (GRand Unified Bootloader) is a menu-based bootloader that replaced LILO. Version 1 (Grub Legacy) appeared in 1999, and version 2, a complete rewrite, appeared in 2005. It allows selecting kernels or chainloaders and editing boot options on the fly**
**فارسی: **GRUB (بوت‌لودر یکپارچه) جایگزین LILO شد. نسخه ۱ (Grub Legacy) در ۱۹۹۹ و نسخه ۲ بازنویسی کامل آن در ۲۰۰۵ منتشر شد. با GRUB می‌توان کرنل‌ها یا chainloaderها را انتخاب کرد و گزینه‌های بوت را به‌صورت لحظه‌ای ویرایش نمود

## Grub Legacy
English: **GRUB v1 (0.9) is usually installed in /boot/grub. Its main configuration is in /boot/grub/menu.lst, though some modern distros (like RedHat-based) link it to /boot/grub/grub.conf. A GRUB legacy config file has two sections: global settings first, then kernel/initramfs or chainloader entries**
**فارسی: **نسخه ۱ GRUB (۰.۹) معمولاً در /boot/grub نصب می‌شود. فایل پیکربندی اصلی آن در /boot/grub/menu.lst قرار دارد، اگرچه در برخی توزیع‌های جدید (مثل RedHat) به /boot/grub/grub.conf لینک شده است. فایل پیکربندی GRUB Legacy دو بخش دارد: ابتدا تنظیمات کلی و سپس گزینه‌های کرنل/initramfs یا chainloader

## GRUB2 
English: **GRUB2 is the most common bootloader today. On BIOS systems, it’s installed in /boot/grub/ or /boot/grub2/, and on UEFI systems, in /boot/efi/EFI/distro-name/ (e.g., /boot/efi/EFI/fedora/). Its configuration file is grub.cfg. GRUB2 uses Linux-style partition numbering: the first partition on the first disk is (hd0,1) (or (hd0,msdos1) for DOS partitions, (hd0,gpt1) for GPT)**
