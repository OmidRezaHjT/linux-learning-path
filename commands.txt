# Commands

## man 
"man [command]"
English: **displays the manual page for a command, showing usage, options, and descriptio**
**فارسی: **این دستور صفحه راهنمای یک دستور را نمایش می‌دهد و شامل نحوه استفاده، گزینه‌ها و توضیحات آن است

## ls
"ls [options]"
**English:** List files and directories in the current folder.  
**فارسی:** لیست گرفتن از فایل‌ها و دایرکتوری‌های موجود در پوشه فعلی.

## dmesg
"dmesg [options]"
English: **dmesg shows kernel messages but does not keep the boot process log permanently; you can use journalctl -b to view boot logs.**
**فارسی: **دستور dmesg پیام‌های کرنل را نشان می‌دهد اما لاگ پروسس بوت را نگه نمی‌دارد؛ برای دیدن لاگ بوت می‌توان از journalctl -b استفاده کرد

## pstree
"pstree [options]"
English: **pstree shows running processes in a tree-like structure, displaying parent-child relationships**
**فارسی: **این دستور فرآیندهای در حال اجرا را به صورت درختی نشان می‌دهد و رابطه والد و فرزند بین آن‌ها را نمایش می‌دهد

## systemctl
"systemctl [command]"
English: **systemctl is the main command to manage services in systemd (start, stop, enable, disable, check status)**
**فارسی: **دستور systemctl فرمان اصلی برای مدیریت سرویس‌ها در systemd است (اجرا، توقف، فعال‌سازی، غیرفعال‌سازی، بررسی وضعیت)

## systemctl list-units 
"systemctl list-units [options]"
English: **Lists all active units (services, sockets, targets, etc.) managed by systemd**
**فارسی: **لیست تمام یونیت‌های فعال (سرویس‌ها، سوکت‌ها، تارگت‌ها و …) که توسط systemd مدیریت می‌شوند

## systemctl get-default
"systemctl get-default"
English: **Shows the default target, i.e., which group of services the system starts by default (like multi-user or graphical)**
**فارسی: **تارگت پیش‌فرض سیستم را نشان می‌دهد؛ یعنی گروهی از سرویس‌ها که سیستم به طور پیش‌فرض اجرا می‌کند (مثلاً multi-user یا graphical)

## systemctl is-system-[state]
"systemctl is-system-[state]"
English: **checks the overall state of the system. You replace [state] with one option at a time (running, degraded, maintenance, initializing, starting, stopping) to see if the system is in that specific state**
**فارسی: **این دستور وضعیت کلی سیستم را بررسی می‌کند. [state] را هر بار با یکی از گزینه‌ها (running، degraded، maintenance، initializing، starting، stopping) جایگزین می‌کنید تا مشخص شود سیستم در آن وضعیت خاص است یا خیر

## journalctl
"journalctl [options]"
English: **journalctl shows logs collected by systemd’s journal, including system, kernel, service, and boot logs (e.g., journalctl -k for kernel, journalctl -b for boot)**
**فارسی: **دستور journalctl لاگ‌های جمع‌آوری‌شده توسط journal در systemd را نمایش می‌دهد، از جمله لاگ‌های سیستم، کرنل، سرویس‌ها و بوت (مثلاً journalctl -k برای کرنل و journalctl -b برای بوت)

## runlevel
"runlevel"

## telinit
"telinit [runlevel]"
English: **you can changes the current runlevel of a SysVinit system, effectively switching the system to a different stage of operation.**
**فارسی: **دستور telinit runlevel فعلی سیستم SysVinit را تغییر می‌دهد و سیستم را به مرحله‌ای دیگر از عملکرد منتقل می‌کند.
Note: /etc/inittab is used by SysVinit. systemd does not use this file ; it uses targets instead (e.g., default.target)

**
**فارسی: **دستور shutdown برای خاموش یا ریبوت امن یک سیستم لینوکس استفاده می‌شود و می‌توان کاربران را مطلع کرد؛ همچنین می‌تواند halt (توقف تمام پردازش‌ها و CPU) را نیز انجام دهد

**فارسی: **یک فایل متنی که قبل از نمایش صفحه ورود نشان داده می‌شود و معمولاً برای نمایش اطلاعات سیستم یا هشدارهای قانونی استفاده می‌شود

## parted
parted [command]
English: **A command-line tool for managing disk partitions, allowing you to create, delete, resize, move, and check partitions**
**فارسی: **یک ابزار خط فرمان برای مدیریت پارتیشن‌های دیسک است که امکان ساخت، حذف، تغییر اندازه، جابجایی و بررسی پارتیشن‌ها را فراهم می‌کند

Note: parted does not understands GPT

## fdisk (BIOS)
fdisk [command]
English: **A command-line utility to view and manage disk partitions, mainly on MBR (Master Boot Record) partitioned disks. It can create, delete, resize, and list partitions**
**فارسی: **یک ابزار خط فرمان برای مشاهده و مدیریت پارتیشن‌های دیسک است که بیشتر روی دیسک‌هایی با جدول پارتیشن MBR استفاده می‌شود. امکان ساخت، حذف، تغییر اندازه و لیست کردن پارتیشن‌ها را دارد

## gdisk (UEFI)
gdisk [command]
English: **A command-line utility used to view and manage partitions on GPT (GUID Partition Table) disks. It is similar to fdisk but designed for modern GPT-based disks, supporting larger disk sizes and more partitions**
**فارسی: **یک ابزار خط فرمان برای مشاهده و مدیریت پارتیشن‌ها روی دیسک‌هایی با جدول پارتیشن GPT است. مشابه fdisk عمل می‌کند ولی مخصوص دیسک‌های جدید با GPT است که از حجم‌های بزرگ‌تر و تعداد بیشتری پارتیشن پشتیبانی می‌کنند

## gparted
English: **GParted (GNOME Partition Editor) is a graphical partition manager for Linux. It lets you create, delete, resize, move, copy, and check partitions and file systems with an easy-to-use GUI, instead of command-line tools like fdisk, parted, or gdisk**
**فارسی: **جی‌پارتد یک مدیر پارتیشن گرافیکی در لینوکس است. با آن می‌توان به راحتی و با محیط گرافیکی پارتیشن‌ها و فایل‌سیستم‌ها را ساخت، حذف کرد، تغییر اندازه داد، جابه‌جا کرد، کپی گرفت یا بررسی کرد

## LVM
English: **LVM (Logical Volume Manager) is a system in Linux that allows flexible disk management. Instead of working directly with partitions, you can create logical volumes on top of physical disks. With LVM, you can easily resize partitions, combine multiple disks into one volume, take snapshots, and manage storage more dynamically
PV (Physical Volume): A whole disk or a partition. It’s better to use partitions rather than raw disks.
VG (Volume Group): A collection of one or more PVs. The OS sees it as one big disk. PVs can differ in size or be on different physical disks.
LV (Logical Volume): Appears as a partition to the OS. It can be formatted and used like a regular partition.
**
**فارسی: **مدیریت حجم منطقی در لینوکس، سیستمی برای مدیریت انعطاف‌پذیر دیسک‌ها است. به جای کار مستقیم با پارتیشن‌ها، می‌توان روی دیسک‌های فیزیکی حجم‌های منطقی ساخت. با LVM می‌توان راحت‌تر پارتیشن‌ها را تغییر اندازه داد، چند دیسک را یکی کرد، از حجم‌ها snapshot گرفت و مدیریت فضای ذخیره‌سازی را ساده‌تر انجام داد

/etc/fstab : The /etc/fstab file (File System Table) is a configuration file in Linux/Unix that defines how disk partitions, devices, or remote file systems should be automatically mounted at boot. Each line specifies a device, mount point, filesystem type, mount options, dump, and fsck order
