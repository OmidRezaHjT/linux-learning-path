# Commands

## man
"man [command]"
English: **Displays the manual page for a command, showing usage, options, and description**
Finglish: **In dastoor safhe-ye rahnamaye yek dastoor ra namayesh mide va shamel nahve estefade, gozine-ha va tozihat an ast**

## ls
"ls [options]"
English: **List files and directories in the current folder**
Finglish: **List gereftan az file-ha va directory-ha-ye mojood dar poshe-ye feli**

## lsusb
"lsusb [options]"
English: **List all USB devices connected to the system**
Finglish: **List tamame dastgah-ha-ye USB motasel be system**

## lspci
"lspci [options]"
English: **List all PCI devices (like graphics cards, network cards)**
Finglish: **List tamame dastgah-ha-ye PCI (mesl-e card-e grafik, card-e shabake)**

## lsblk
"lsblk [options]"
English: **List information about all block devices (disks, partitions)**
Finglish: **Namayesh etela’at hame-ye dastgah-ha-ye block (hard, partition-ha)**

## lshw
"lshw [options]"
English: **Display detailed hardware information of the system**
Finglish: **Namayesh etela’at sakht-afzari daghigh-e system**

## lsmod
"lsmod"
English: **Shows kernel modules. They are located at /lib/modules**
Finglish: **Neshan dadan module-ha-ye kernel**

## cat

Finglish: **Namayesh mohtavaye yek file**

## rm

## echo

Finglish: **Chap matn dar terminal ya neveshtan dar file**

## insmod

Finglish: **Ezafe kardan yek module kernel be system**

## rmmod
"rmmod [module]"
English: **Remove a kernel module from the system**

## modinfo
"modinfo [module]"
English: **Show information about a kernel module (version, dependencies, author, etc.)**
Finglish: **Namayesh etela’at darbare-ye yek module kernel (version, dependencies, author, ...)**

## dmesg
"dmesg [options]"
English: **dmesg shows kernel messages but does not keep the boot process log permanently; you can use journalctl -b to view boot logs**
Finglish: **Dastoor dmesg payam-ha-ye kernel ra neshan mide vali log-e boot ra hefz nemikonad; baraye didan log-e boot mitavan az journalctl -b estefade kard**

## pstree
"pstree [options]"
English: **pstree shows running processes in a tree-like structure, displaying parent-child relationships**
Finglish: **In dastoor farayand-ha-ye dar hal ejra ra be soorat-e derakhti neshan mide va rabete-ye valed-va-farzand ra namayesh mide**

## systemctl
"systemctl [command]"
English: **systemctl is the main command to manage services in systemd (start, stop, enable, disable, check status)**
Finglish: **Dastoor systemctl farman asli baraye modiriat-e service-ha dar systemd ast (ejra, stop, enable, disable, check status)**

## systemctl list-units
"systemctl list-units [options]"
English: **Lists all active units (services, sockets, targets, etc.) managed by systemd**
Finglish: **List tamame unit-ha-ye faal (service-ha, socket-ha, target-ha, ...) ke tavasot-e systemd modiriat mishavand**

## systemctl get-default
"systemctl get-default"
English: **Shows the default target, i.e., which group of services the system starts by default (like multi-user or graphical)**
Finglish: **Target pishfarz system ra neshan mide; yani gorohi az service-ha ke system be tore pishfarz ejra mikonad (mesl-e multi-user ya graphical)**

## systemctl is-system-[state]
"systemctl is-system-[state]"
English: **Checks the overall state of the system. Replace [state] with one option at a time (running, degraded, maintenance, initializing, starting, stopping)**
Finglish: **In dastoor vaziyat-e kolli system ra barrasi mikonad. [state] ra har bar ba yek gozine jaygozin konid (running, degraded, maintenance, initializing, starting, stopping)**

## journalctl
"journalctl [options]"
English: **Shows logs collected by systemd’s journal, including system, kernel, service, and boot logs (journalctl -k for kernel, journalctl -b for boot)**
Finglish: **Dastoor journalctl log-ha-ye jamavari shode tavasot-e journal dar systemd ra neshan mide, az jomle log-ha-ye system, kernel, service va boot**

Finglish: **Modir-e partition grafiki dar Linux ke ejad, hazf, taghir andazeh, jabja, copy va barrasi partition-ha va file system ra ba GUI anjam midahad**

## LVM
"lvm [command]"
English: **Logical Volume Manager: flexible disk management using PV, VG, LV

PV (Physical Volume): A whole disk or a partition. It’s better to use partitions rather than raw disks.
VG (Volume Group): A collection of one or more PVs. The OS sees it as one big disk. PVs can differ in size or be on different physical disks.
LV (Logical Volume): Appears as a partition to the OS. It can be formatted and used like a regular partition.
**

Finglish: **Modiriat-e mojaz-e volume dar Linux, estefade az PV, VG, LV baraye modiriat-e fleksibel-e disk**

## ld or ld-linux
"ld|ld-linux [options] [executable]"

Finglish: **Baraye ejra-ye dastkari-ye barname ba dynamic linker an, va load kardan tamame ketabkhane-ha-ye eshteraki ghabl az ejra. Mamulan niyaz nist farakhani shavad chon ELF executables khodeshun ro moshakhas mikonand**
